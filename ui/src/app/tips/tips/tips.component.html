• DO NOT change the class name. Class name should always be Main, since that is used to load the class.
  • DO NOT remove imports set by default. They need to be there in order to compile the annotations properly.<
  • Bytecode should be written in the following format:
    /*
    <bytecode>
    */
    Comment start and end symbols (/* and */) should be written on separate lines. This is referred to as a Jasm block.
  • Methods containing Jasm blocks should be annotated with @Jasm. @Jasm annotation accepts a list of @Block annotations. @Block annotation accepts 2 parameters: start and end, denoting the line where the Jasm block starts and ends. Value of start and end should be the line, where the actually bytecode begins and ends (not where the comments begin and end). There should be a @Block annotation for each Jasm block in a given method.
